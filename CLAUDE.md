# CLAUDE.md — go-llm-cli-kit

このリポジトリは **社内標準（`LLM.md`）** と、それを Go CLI に実装するための
**共有モジュール** と **雛形（`templates/`）** を同居させている。
15本の CLI がここを import しているため、変更は全社に波及する。

## 変更時の必須手順

**機能を追加した、API を変えた、既定の挙動を変えた — このいずれかをしたら、
以下をすべて更新してから終わること。**

| 更新先 | いつ |
| --- | --- |
| パッケージの doc コメント | 公開 API を足した・変えた |
| `README.md` | 使い方・パッケージ構成が変わった |
| **`LLM.md`**（標準本体） | 方針・規約・チェックリストに影響する変更 |
| **`templates/`** | 各 CLI にコピーされる内容が変わった |
| テスト | 常に。特に `skillcheck` は雛形自身を検証している |

`templates/` の更新を忘れると、**次に横展開した CLI だけが古い形になる**。
実装を変えたら雛形も直す、が原則。

雛形の SKILL.md / plugin.json を変えたら `go test ./...` を必ず走らせること。
`TestShippedTemplatePasses` が雛形自身を `skillcheck` に通しており、壊れた雛形が
15本に伝播するのを防いでいる。

## 標準そのものを変えるとき

`LLM.md` は横展開の基準であり、既に適用済みの CLI がある。方針を変える場合は:

1. `LLM.md` の該当節を直す
2. 変更の理由を残す（「当初こうしたが、実装したらこうだった」の形が最も役に立つ）
3. 既に適用済みの CLI に遡及が必要かを判断し、必要なら issue に起こす

パイロット（vultr-cli）で判明した差し戻しは §5-3 / §5-3-1 / §10 に反映済み。
同じ形で追記していく。
